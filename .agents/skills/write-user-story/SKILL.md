# Skill: Escrita de User Stories (INVEST)

Esta skill auxilia na transformação de necessidades de negócio em User Stories bem estruturadas que servem como base para o desenvolvimento.

## 🏗️ Estrutura da User Story

O formato padrão deve ser:

**"Como [persona/papel], eu quero [ação/funcionalidade], para que [valor/benefício]."**

### Componentes:
- **Papel**: Quem se beneficia da funcionalidade? (ex: Pescador, Guia, Administrador).
- **Ação**: O que o usuário está tentando realizar?
- **Valor**: Por que isso é importante? Qual problema resolve ou qual oportunidade aproveita?

---

## 💎 O Padrão INVEST

Uma boa User Story deve ser:

1.  **I**ndependent (Independente): Deve ser possível desenvolvê-la sem depender de outras histórias.
2.  **N**egotiable (Negociável): Não é um contrato rígido; detalhes devem ser discutidos entre o time e o PO.
3.  **V**aluable (Valiosa): Deve entregar valor claro para o usuário ou para o negócio.
4.  **E**stimable (Estimável): O time deve ser capaz de estimar o esforço necessário.
5.  **S**mall (Pequena): Deve ser pequena o suficiente para ser concluída em uma iteração (Sprint).
6.  **T**estable (Testável): Deve ser possível verificar se a história foi concluída com sucesso.

---

## 🚀 Melhores Práticas

- **Foque no Usuário**: Evite termos excessivamente técnicos no título e na descrição da história.
- **Voz Ativa**: Use verbos de ação (ex: "verificar", "agendar", "filtrar").
- **Inclua Critérios de Aceitação**: Uma história sem critérios de aceitação está incompleta.
- **Decomposição**: Se uma história for muito grande (Epic), divida-a em histórias menores.
